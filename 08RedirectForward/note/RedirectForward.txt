
Redirect
  server 측에서 client 측으로
  request address(URL)를 보내주면 (response)
  client 의 browser가 이를 받아서
  다시 server로 request(address:URL)를 보냄
    ㄴ client 의 browser가 server 측으로
    request를 보낸다는 것을 인식함
      ㄴ browser 의 주소창에는 이 주소(URL)가 표시 됨.
      
  Redirect 방식에서는 새로운 request 가 발생하므로
  HttpServletRequest 객체가 메모리에서 삭제가 되었다가 다시 생성됨.
      
Forward
  server 측에서 client 측으로
  request address(URL)를 보내주지 (response) 않음.
    ㄴ client 의 browser는 page 가 이동하는지 모름.
  	ㄴ browser 의 주소창에는 이 주소(URL)가 표시 됨.
